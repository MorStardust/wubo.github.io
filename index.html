hexo+gitpage搭建博客

Posted on 2019-09-11 Edited on 2019-09-16

概述

  为什么我要用 Hexo + GitPage 搭建博客？为什么要用Hexo,起初我并不了解，带着疑惑我一步步搭建完博客之后，我有所总结。所以先告诉大家为什么，然后再来说怎么做。
-Hexo
  Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，使用 Node.js 渲染页面，在几秒内，即可利用靓丽的主题生成静态网页。
-GitPage
  GitPage 允许你将你的博客创建为一个 GitHub Project，通过 your-account.github.io 这样的特殊项目名称与 GitPage 进行关联，然后，你只需要像平时一样 commit 你的博文到 GitHub 上就 OK 了，GitPage 会自动将你的更新部署出去，完全不需要考虑服务器、数据库、运维、甚至 HTML 等这些发布站点的必备技能，所以 GitPage 诞生的目的就是要把你的专注点拉回到写文章上而不需要花时间去考虑其它事情。
-两者结合Hexo+GitPage
  利用hexo本地编写博客搭建框架，发布到gitpage上。现实中遇到新的技术，项目，它们需要的文档并不是写几个 Readme 就能讲清楚的，这种情况下就急需一个站点来详尽展示此技术的功能、用法、及更新动态等等，认真做好博客，相信你我都会受益于此。

搭建 Node.js 环境

  Node.js 官网：https://nodejs.org/en/下载
保持默认设置即可，一路Next，安装很快就结束了。然后打开命令提示符，输入node -v、npm -v，出现版本号则说明 Node.js 环境配置成功

搭建 Git 环境

  git 下载 https://pc.qq.com/detail/13/detail_22693.html
下载安装包桌面右键，打开Git Bush Here，输入git –version，出现版本号则说明 Git 环境配置成功，第二步完成

GitHub 注册和配置

  Github注册：https://github.com/创建仓库：Repository name 使用自己的用户名，命名规则为yourname.github.io，访问 yourname.github.io，如果可以正常访问，那么 Github 的配置已经结束了。

注意

  出现404 在Settings中找到github pages 将source改成master branch 当提示Your site is published at https://××××.github.io/×××.github.io/即为成功，出现提示CNAME命名不规范在<>Code中删除CNAME

安装配置 Hexo

  在命令行cmd中输入npm install hexo-cli -g 在桌面右击进入gitbash，输入hexo version查看Hexo的版本，在电脑中建立一个名为【hexo】的文件夹，新建的文件夹右击打开gitbash，输入hexo init 命令，接着输入hexo server，您的网站会在http://localhost:4000下启动。如果http://localhost:4000能够正常访问，则说明 Hexo 本地博客已经搭建起来了,按ctrl+C可以停止server.

注意1：执行hexo server提示找不到该指令

  解决办法：在Hexo 3.0 后server被单独出来了，需要安装server，安装的命令如下：在桌面右击进入gitbash，输入npm install hexo -server –save

接下来新打开一个git bash命令行窗口，cd到/D/hexo下，执行 hexo new “My New Post” 命令,可以生成一篇新的文章 “My New Post”，打开http://localhost:4000/ 可以预览到新生成的文章。

执行 hexo generate命令可以将markdown文章生成静态网页，该命令执行完后，会在 D:\Hexo\public 目录下生成一系列html，css等文件。

写文章

利用 Github网站,可以轻松管理编写自己的博客,文章统一放在网站根目录下的 _posts 的文件夹中

格式

每一篇文章文件命名采用的是2019-09-10-Hello-2019.md时间+标题的形式，空格用-替换连接。文件的格式是 .md 的 MarkDown 文件。开头大概格式是

 ---
layout:     post                    # 使用的布局（不需要改）
title:      My First Post               # 标题 
subtitle:   Hello World, Hello Blog #副标题
date:       2019-09-10              # 时间
author:     WB                      # 作者
header-img: img/post-bg-2015.jpg    #这篇文章标题背景图片
catalog: true                       # 是否归档
tags:                               #标签
    - TAG
---

按格式创建文章后，提交保存。进入你的博客主页，新的文章将会出现在你的主页上.
